GAP and FC

在卷积网络的最后，通常会有一两层全连接层。全连接层的作用是把卷积输出的二维特征图转化成一个一维的向量，方便网络最后为分类做结果的输出。

全连接层的特点是，每一个节点都与上一层的节点连接，把前一层的输出特征都综合起来。通常，该层的权值参数是最多的。例如在VGG16中，第一个全连接层FC1中有4096个节点，上一层的POOL2的输出是7*7*512=25088个节点，那么需要的权值的数量是4096*25088个，将耗费非常大的内存。

以下图为例：

最后20*12*12的特征图是如何变成1*100的向量的呢？靠的就是全连接层！

依靠20*100个12*12的卷积核卷积出来的，即对于每一个输入的12*12，用一个相同大小的核进行卷积操作，得到一个数。如果输入的深度是20，那么就用20个卷积核卷积后再相加。这样就把一张带深度的图变成了一个数字。

但是，全连接层的参数实在太多了！以上面的讨论为例，存在20*12*12*100个权重参数。

通常会使用全局平均池化 层（GAP）来代替全连接层（FC）。

GAP 的基本思想是，用 feature map 直接表示属于某个类的 confidence map。假如有10个类，就在最后输出10个 feature map，每个 feature map 的值加起来求平均值，这10个数字就对应相应类别的概率或者置信度。然后把得到的平均值作为某个类别的 confidence value，在输入 softmax 中进行分类。实验验证，GAP 的效果并不比 FC 差。

具体参数量的对比：

FC : 全连接先把 feature map 压缩成一列，然后接两层全连接层。首个全连接层对应的权重参数量为：4*4*4*4，第二个全连接层对应的权重参数量为4*4。

GAP：在得到 confidence value 的过程中，不存在权重参数，后面的全连接层对应的权重参数量为4*4。

GAP 的优势是：

  1. 避免了 FC 参数数量巨大的问题，有助于压缩模型的大小。（GAP 没有参数，可以防止该层过拟合。）
  2. 可以接受任意尺寸的图像。
  3. 可以更好地将类别与最后一个卷积层的特征图对应起来。每一个通道对应一种类别，这样每一张特征图都可以看成该类别对应的类别置信图。
